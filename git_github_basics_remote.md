# GitHub Website Beginner Tips

## Assumptions
- You have a GitHub account
- You are logged in at https://github.com
- You are using a desktop browser

---

## 1. Profile Setup (DO THIS FIRST)
- Click your avatar ‚Üí **Settings**
- Add:
  - Profile picture
  - Name
  - Bio
- Verify email

WHY:
- Commits show your identity
- Required for contributions to appear

---

## 2. Create a Repository
Steps:
1. Click **+** (top-right) ‚Üí New repository
2. Enter repository name
3. Choose:
   - Public (visible to everyone)
   - Private (only you / collaborators)
4. Check **Add a README.md**
5. Click **Create repository**

TIP:
- Always add a README for new repos

---

## 3. Repository Page Basics
Important tabs:
- **Code** ‚Üí files and folders
- **Issues** ‚Üí bug reports, tasks
- **Pull requests** ‚Üí code reviews
- **Actions** ‚Üí CI/CD
- **Settings** ‚Üí repo configuration

---

## 4. README.md (VERY IMPORTANT)
Purpose:
- Explain what the project is
- How to run it
- How to contribute

Minimum README structure:
- Project name
- Description
- Setup steps
- Usage
- License

---

## 5. Upload Files from Website
Steps:
1. Open repo ‚Üí **Add file**
2. Choose **Upload files**
3. Drag files
4. Commit changes

TIP:
- Good for small edits only
- Use Git CLI for real work

---

## 6. Editing Files Online
Steps:
1. Open file
2. Click **‚úèÔ∏è Edit**
3. Make changes

TIP:

---

## 7. Commits on Website
- Every website change creates a commit
- Write meaningful commit messages
- Choose correct branch before committing

---

## 8. Branches (Website)
Steps:
1. Click branch dropdown
2. Type new branch name
3. Press Enter

TIP:
- Use branches for features and experiments

---

## 9. Pull Requests (PR)
Purpose:
- Merge one branch into another
- Review code before merging

Steps:
1. Push branch
2. GitHub shows **Compare & pull request**
3. Add title + description
4. Create PR
5. Merge

TIP:
- Always describe WHAT and WHY

---

## 10. Issues (Task Tracking)
Use Issues for:
- Bugs
- Feature requests
- TODOs

Steps:
1. Open **Issues**
2. New issue
3. Add title + description
4. Submit

TIP:
- One issue = one problem

---

## 11. Forking Repositories
Meaning:
- Create your own copy of someone else's repo

Used for:
- Open-source contributions

Flow:
Fork ‚Üí Clone ‚Üí Branch ‚Üí Commit ‚Üí PR

## 13. Releases
Purpose:
- Mark stable versions
- Attach build files

Used for:
- Libraries
- Apps
- Versioning

---

## 14. Security Basics
- NEVER commit:
  - Passwords
  - Tokens
  - API keys
- Use `.gitignore`

---

## 15. Common Beginner Mistakes
- Pushing secrets
- Working on `main` directly
- Huge commits
- No README
- No commit messages

---

## 16. Golden Rules
- README first
- Small commits
- Use branches
- Pull before push
- Never force-push to shared branches
